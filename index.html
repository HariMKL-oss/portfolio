02 — Featured Project

LabelSentinel

Automated Nutritional Intelligence Platform

🚀 Live Demo 📋 Patent Pending 🕶️ AR-Ready

An AI-powered food safety auditing system designed for Meta's AR smart glasses. The user looks at a product and says "Scan this" — the AI reads the ingredient label and projects a verdict directly into their field of view.

Live Demo ↗ GitHub ↗
🔴

Problem

Consumers spend 3-5 minutes per product deciphering dense ingredient labels. 80% of packaged foods contain hidden inflammatory oils, synthetic additives, or misleading "natural" labels. There's no instant, hands-free way to audit food safety.

🟢

Solution

A multimodal LLM orchestration layer integrating Gemini 2.5 Pro, GPT-4o, and on-device SLMs (Llama 3.2 3B, Gemma 3 4B) with automatic tier escalation based on OCR confidence thresholds. Real-time OCR + NLP pipeline using ML Kit Text Recognition V2 for ingredient label extraction gets >90% per-word accuracy.

Three-Tier Inference Architecture

Tier 1 < 500ms

On-Device Fast Path

ML Kit OCR V2 Keyword Matcher Barcode Scanner (UPC/EAN)

Handles ~70% of products — zero network calls, offline capability

↓
Tier 2 1–3s

On-Device SLM Reasoning

Gemma 3 4B Llama 3.2 3B

Resolves ambiguity without cloud dependency based on confidence thresholds

↓
Tier 3 2–5s

Cloud LLM Deep Analysis

Gemini 2.5 Pro GPT-4o

Complex labels. Optimized for latency (<1.5s) and cost ($0.004/call).

06 — Philosophy

How I think about AI engineering.

01

Reliability Over Novelty

The best AI system is the one that works at 3 AM without paging anyone. I prioritize deterministic fallbacks, graceful degradation, and observability over chasing the latest model.

02

Hybrid Inference Architecture

Not every query needs GPT-4. I design tiered systems where simple tasks run on-device, moderate tasks use quantized SLMs, and only complex queries hit cloud LLMs. Cost and latency optimized by design.

03

Agents That Ship

Agentic architectures are powerful but fragile. I build agent systems with structured outputs, tool validation, retry logic, and human-in-the-loop checkpoints — not autonomous chaos.

04

Infrastructure as AI Enabler

The gap between a Jupyter notebook and a production API is massive. I close that gap with containerized deployments, CI/CD for models, feature stores, and monitoring that actually catches drift.

05

Cost-Conscious AI

Enterprise AI must be cost-sustainable. I design systems that minimize token usage, cache aggressively, batch intelligently, and know when a regex beats a transformer.

06

Products, Not Demos

I don't build proof-of-concepts that collect dust. Every system I create has a deployment path, monitoring, error handling, and a user who depends on it.
